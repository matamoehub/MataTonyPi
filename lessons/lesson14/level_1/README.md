# Lesson 14 - Multi-Control Robot Team

## Goal

Set up one TonyPi as the server robot and the others as client robots, then test synchronized movement and action groups.

## What students will use

- robot-console live TonyPi list
- robot-console multi-control configuration
- TonyPi action groups through `myRobot.anim.run(...)` and `myRobot.anim.run_id(...)`
- simple synchronized movement checks like wave, bow, and dance

## Teaching Note

This lesson depends on the robot-console TonyPi multi-control API and the upstream TonyPi server/client scripts. After applying a new server/client configuration, restart the TonyPi runtime or reboot the robots before testing.
